Tiny x86 OS — In‑Browser Runner

Write in assembly, build with NASM, then boot your raw image here — right in your webpage.

How to use

  1. Click Download sources to get boot.asm, kernel.asm, and Makefile.
  2. Install NASM & QEMU (e.g., sudo apt-get install nasm qemu-system-x86).
  3. Run make to produce os.img (raw 1–2 sector image).
  4. Upload os.img and click Run in browser.

Expected output: BIOS boots the 512‑byte boot sector, prints Booting tiny OS…, loads the kernel sector, and the kernel writes to the VGA text buffer.

Controls

Status: Idle

Emulator

Upload an os.img and click Run

Notes & Tips

  • Ensure kernel.bin is exactly 512 bytes for this demo (we use truncate -s 512). You can extend to multiple sectors later.
  • You can do a single‑sector OS entirely in the boot sector and skip loading kernel.bin.
  • Windows: use WSL or install NASM + QEMU via Chocolatey (choco install nasm qemu).
  • v86 emulates a legacy BIOS PC. Raw floppy‑style images work well.
